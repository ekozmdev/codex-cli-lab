# AGENTS.md

## リポジトリ概要
OpenAI codex-cliに関する情報や、現在の個人設定を記録するリポジトリ。このAGENTS.mdはクラウドベースのCodexから利用することを想定する。

## リポジトリ構成

```plain
codex-cli-lab
├── AGENTS.md
├── README.md
├── my-config/ # ホームのCodex CLI設定
├── reports/ # レポート置き場
└── update-config.sh # ローカル設定更新用スクリプト
```

## クラウドCodexでの利用手順

1. 追加の言語セットアップは不要。
2. https://github.com/openai/codex/releases で最新リリース情報を確認する（alpha/betaなどプレリリースは除外してよい）。
3. READMEに記載されたバージョンより新しい正式リリースが存在する場合のみ、以下のレポート作成を実施する。
4. 最新バージョンの情報がない場合は何も変更せず、作業を終了する

## レポート作成フロー

1. `https://github.com/openai/codex.git` を一時ディレクトリ配下にチェックアウトする。
2. タグ一覧から最新タグと1つ前のタグを確認する（alpha/betaなどのプレリリースタグはスキップする）。
3. `git diff` で最新タグと1つ前のタグを比較し、差分を確認する。
4. 新機能や新しい設定項目など、重要な変更点を抜粋してまとめる（細かいバグ修正に関する情報は不要）。
5. `reports/yyyy-MM-dd-x-y-z.md` というMarkdownファイル（yyyy-MM-ddはリリースされた日付、`x-y-z` はバージョン）を作成し、レポートを保存する。公式タグに `rust-` などの接頭辞が付いていても、ファイル名のバージョン表記には含めない。

## レポートの形式

```markdown

## 追加された機能・設定項目

## 廃止された(または廃止予定の)機能・設定項目

## その他

```

## READMEの更新
レポート作成後、READMEのバージョン情報も更新する。

例)

```
- 最新バージョンは`x.y.z`です(yyyy/MM/ddリリース)
```

## 作業後の確認
- 作成・更新したレポートとREADMEの差分を確認し、誤記や不足がないかレビューする。
- 不要な一時ファイルやデバッグコードが残っていないか再確認してからコミット・プッシュする。
